# /checkup — 자라는 CLAUDE.md (재문진)

Seed 의 출생은 인터뷰(`/onboard`)다. 하지만 워크스페이스는 계속 자라고, 초기 인터뷰의 답은 낡는다. `/checkup` 은 축적된 데이터를 읽어 **워크스페이스가 스스로를 가볍게 다시 문진**하고, CLAUDE.md 를 지금의 나에 맞추도록 돕는다.

## 절차

1. **관측 수집** — `python bin/ws.py checkup` 을 실행한다. 주요 도메인·자주 붙인 태그·가장 엮인 지식(씨실)·산출물/지식/결정 수를 보여주고, 마지막 재문진 기록을 오늘로 갱신한다.
2. **대조** — 출력된 관측을 **현재 CLAUDE.md** 와 비교한다. 특히 `주요 도메인`·`주 산출물 형태`·조력 에이전트가 실제 사용과 어긋나는지 본다.
3. **제안** — 어긋난 부분을 사용자에게 **제안**한다. 예: "요즘 `정산`·`운영` 업무가 많은데 CLAUDE.md 주요 도메인엔 `기획`만 있어요 — 갱신할까요?" / "`spec` 템플릿은 한 번도 안 쓰였어요 — 접어둘까요?"
4. **승인 후 편집** — 사용자가 동의한 것만 CLAUDE.md·`templates/`·`seed.json` 에 반영한다.

## 트리거 (어떻게 재문진 때가 왔음을 아나)

Seed 는 의존성 0·정적이라 데몬·cron 이 없다. 대신 **어시스턴트가 heartbeat** 다:

- **세션 시작 신호** — 워크스페이스 작업을 시작할 때 `python bin/ws.py checkup --due` 로 시점인지 확인한다(기준: 마지막 재문진 30일 경과 또는 산출물 15건 누적). due 면 사용자에게 **한 번만** 권한다. 아니면 조용히 넘어간다.
- **대시보드 칩** — 재문진 시점이면 대시보드 상단에 `🌱 재문진 권장 · /checkup` 칩이 뜬다.

## 규율 (재문진을 재문진으로 유지)

- **강요하지 않는다.** 권하되, 사용자가 넘기면 그걸로 끝. 매 세션 조르지 않는다(마커가 갱신되므로 자연히 조용해진다).
- **CLAUDE.md 를 자동으로 고치지 않는다.** 도구는 관측만 주고, 편집은 사람의 승인을 거친다.
- 재문진은 **덜어내는 자리**이기도 하다 — 안 쓰는 템플릿·낡은 도메인·죽은 조력 에이전트를 접는 것도 성장이다.
