# /decide — 결정 기록

무엇을 정했는지 + **왜** 정했는지를 남긴다. 이유가 결정보다 오래 값지다. (DB 미초기화면 먼저 `python bin/ws.py init`)

## 매핑

| 의도 | 명령 |
|---|---|
| 기록 | `python bin/ws.py decide "무엇을 정했나" --why "왜" [--task ID]` |
| 최근 조회 | `python bin/ws.py decide [--limit N]` |

## 가이드

- 사용자가 "~하기로 했어 / ~로 가자 / 이걸로 확정" 이라고 하면 `decide` 로 남긴다.
- **`--why` 를 최대한 채운다** — 결정만 있고 이유가 없으면 반년 뒤 되돌아볼 때 쓸모가 준다. 사용자가 이유를 말 안 했으면 한 줄로 물어본다.
- 특정 업무에서 나온 결정이면 `--task <id>` 로 묶는다(그 task 가 실제 존재해야 함).
- 인자 없이 `decide` 만 부르면 최근 결정을 이유와 함께 보여준다("왜 그렇게 정했더라").
- 단순 메모는 `/decide` 가 아니라 `/note` 로.
