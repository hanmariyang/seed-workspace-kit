# /links — 씨실 (지식 연결)

지식을 고립된 낱장이 아니라 **그물**로 만든다. `[[주제]]` 하나로 문서끼리 잇고, 무엇이 무엇을 참조하는지 되짚는다. (DB 미초기화면 먼저 `python bin/ws.py init`)

## `[[주제]]` — 잇는 법

wiki·산출물 마크다운 **어디서든** `[[주제]]` 라고 쓰면 그게 곧 링크다. 별도 문법도, 파서도 없다 — 그냥 텍스트에 쌍대괄호를 두르면 씨실이 그걸 grep 으로 엮는다.

```markdown
W27 사고의 원인은 [[정산]] 월확정 stale. [[경계주]] 분할로 해결.
```

## 매핑

| 의도 | 명령 |
|---|---|
| 엮임 현황 (참조 많은 순) | `python bin/ws.py links` |
| 한 주제의 관계 | `python bin/ws.py links "정산"` |

- `links "주제"` 는 **backlinks**(이 지식을 참조하는 것)와 **outlinks**(이 지식이 가리키는 것)를 양쪽으로 보여준다.
- `[[경계주]]` 처럼 아직 지식 문서(`wiki/경계주.md`)가 없어도 링크는 잡힌다 — "채워질 자리"가 드러난다.

## 가이드

- 사용자가 "이거 뭐랑 엮였어 / 관계 보여줘 / 어떤 주제가 핵심이야" 라고 하면 `links` 로 답한다.
- 메모를 `promote` 로 승격할 때 본문에 관련 `[[주제]]` 를 넣어두면 지식이 고립되지 않는다. 승격하며 "이건 `[[X]]` 와 엮을까요?" 하고 제안해도 좋다.
- **절대 하지 않는 것** (씨실을 씨실로 유지하는 규율): `[[]]` 를 파싱해 검증하지 않는다(깨진 링크도 그냥 둔다) · 관계를 그래프·노드로 그리지 않는다(숫자로만) · 백링크를 색인 파일로 저장하지 않는다(매번 즉석 grep). 이 셋 중 하나라도 넘으면 씨실이 아니라 위키 엔진이 된다.
