# /note — 빠른 메모

지나가는 생각·발견·리마인더를 DB 에 즉시 던져 넣는다. (DB 미초기화면 먼저 `python bin/ws.py init`)

## 매핑

| 의도 | 명령 |
|---|---|
| 기록 | `python bin/ws.py note "메모 내용" [--tag 태그]` |
| 최근 조회 | `python bin/ws.py note [--limit N]` |
| **wiki 승격** | `python bin/ws.py promote <note_id> [--topic 주제] [--title 제목]` |

## 가이드

- 사용자가 "적어둬 / 메모 / 기억해둬" 라고 하면 본문을 그대로 `note` 로 넣는다.
- `--tag` 는 선택 — 주제가 뚜렷하면 한 단어로(예: `--tag payroll`), 애매하면 생략.
- 인자 없이 `note` 만 부르면 최근 메모를 보여준다("방금 뭐 적었지").
- 결정(왜 이렇게 했는지)은 `/note` 가 아니라 `/decide` 로 남긴다.

### 메모의 일생 — 상태(DB) → 지식(md)

- **메모는 임시 상태(state)** 다. workspace.db 의 `notes` 에 쌓인다.
- 하나의 지식으로 굳었다 싶으면 **`promote` 로 `wiki/*.md` 지식으로 승격**한다(지식은 플랫 마크다운).
  - 사용자가 "이거 위키로 올려줘 / 정리해서 남겨줘" 하면 `promote <id>` 를 쓴다.
  - `--topic` 은 지식 문서 파일명(`wiki/{topic}.md`) — 같은 주제면 한 파일에 누적된다. 없으면 메모 첫 줄로 슬러그를 만든다.
  - **승격은 단방향**이다: 승격하면 그 메모는 `notes` 에서 이동(삭제)되고 wiki 에만 남는다. 임시 보관소를 비워 두는 게 목적이다.
  - 승격이 좋아 보이면 먼저 "이건 wiki 로 승격해둘까요?" 하고 물어도 좋다.
  - 승격 본문에 관련 주제를 `[[주제]]` 로 이어두면 지식이 고립되지 않고 **씨실**로 엮인다(`/links`).
- 무엇이든 다시 찾을 때는 `/search` (업무·메모·결정·산출물·wiki 통합 검색), 지식끼리의 연결은 `/links`.
