# 보안 정책

## 지원 버전

Seed 는 **템플릿 저장소**입니다. 항상 `main` 의 최신 상태만 유지·지원합니다.

| 버전 | 지원 |
|---|---|
| main (최신) | ✅ |
| 그 외 | ❌ |

## 공격면에 대한 참고

Seed 는 **의존성 0 · 로컬 정적 도구**입니다 — 서버를 띄우지 않고, 네트워크
호출을 하지 않으며, 생성물은 `file://` 로 열리는 정적 HTML 입니다. 따라서
전형적인 웹 취약점(인증·SSRF 등)의 대상이 아닙니다.

그럼에도 다음과 같은 사안은 보안 이슈로 다룹니다:

- 사용자 입력(업무 제목·도메인·산출물 본문)이 생성 HTML 에서 **이스케이프되지 않아
  스크립트가 실행되는 경우** (저장형 XSS 성격)
- 명명 규약(slugify)이 경로를 벗어나 **의도치 않은 파일을 덮어쓰는 경우** (경로 탈출)
- 그 밖에 로컬 파일·데이터를 손상시킬 수 있는 결함
