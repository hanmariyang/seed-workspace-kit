# 사례 001 — 스튜디오 촬영 가이드

> 🌱 **Seed 씨앗 게이트 첫 통과** — 저자(메인테이너)가 아닌 사람이 Seed로 워크스페이스를 꾸려 실제 결과물을 배포한 첫 사례.

## 한 줄 요약

**진영**님이 Seed로 튜터용 *"스튜디오 촬영 가이드"* 를 만들어 공개 배포했다.
🔗 https://jinyoung-sparta.github.io/studio-guide/

## 무엇인가

- **대상**: 스튜디오 촬영을 앞둔 튜터
- **형태**: **채팅형 4문항 사전 점검** — "촬영 준비 같이 확인해볼게요. 질문 4개만 답하면 끝나요." 식으로, 짧은 문항에 답하며 촬영 전 확인 사항을 훑는 구조.
- **진행바**: `0 / 4` — 순서대로 채워 완주하는 체크리스트 UX.

흥미로운 점: 이 가이드 자체가 **Seed 온보딩 인터뷰와 같은 결**(짧은 Q&A → 한 장 정리)이다. Seed로 만든 산출물이 Seed의 상호작용 철학을 그대로 닮았다.

## 왜 중요한가 (씨앗 게이트)

Seed의 성공 정의는 별·클론 수가 아니라 이것 하나다:

> **"나 아닌 한 사람이 복제 → 온보딩 완주 → 산출물 1개."**

| 조건 | 상태 |
|---|---|
| 저자 아닌 사람이 복제 | ✅ 진영 |
| 온보딩 완주 | ✅ |
| 실제 산출물 1개 | ✅ 스튜디오 촬영 가이드 (공개 배포) |

→ **첫 통과 신호 점등.** 다만 아직 *n=1* 이다. 재현(두 번째 통과)과 과정 기록이 쌓여야 "새싹" 단계로 넘어간다.
