## 빌드 규약 (모든 컨셉 공통 — 반드시 지킬 것)

1. `<html lang="ko">`. 모든 UI 텍스트 한국어.
2. `<head>` 에 `<link rel="stylesheet" href="_shared/theme.css">` 한 줄 (빌드 후 자동 인라인됨). 그 외 외부 파일/CDN/웹폰트/JS 라이브러리 금지. 바닐라 JS만, `<script>` 인라인.
3. `.wrap` 최상단에 `<a class="back" href="index.html">← 카탈로그</a>`.
4. `.head` 안에 brand 칩("🌱 민서의 워크스페이스"), `<h1>`, 테마토글 버튼(`<button class="ttog">🌙</button>`), `.stamp`. 토글은 `document.documentElement.dataset.theme` ↔ "dark" 전환 + localStorage "seed-theme" 저장. 페이지 로드 시 저장값 반영.
5. theme.css 의 컴포넌트 클래스를 최대한 재사용(.stat .grid .card .badge .b-* .prio .pbars .dtag(.report/.research/.ops/.plan/.content/.dev) .tag .prog .ring .av(.b/.v/.a/.r/.t) .due .seg .kbd .charts .panel .donut .bars .spark .heat .tl .cal .feed .palette 등). 부족한 것만 컨셉 `<style>` 로 보강.
6. **출처/브랜드명 절대 금지** — 실제 제품·회사·서비스 이름을 산출물 어디에도(주석 포함) 쓰지 말 것. 패턴만 차용.
7. 더블클릭(file://)으로 열려야 함 — 같은 폴더 상대경로만.
8. 인터랙션(검색·필터·정렬·뷰전환·토글 등)은 실제 바닐라 JS로 **작동**해야 함.
