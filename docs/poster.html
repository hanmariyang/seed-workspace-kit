01복사가 아니라 인터뷰

남의 워크스페이스를 통째로 복사하면 내 업무와 안 맞아 방치된다. Seed 는 빈 폴더를 주는 대신 당신을 6~8개 인터뷰하고, 그 답으로 디렉토리·로컬 DB·전용 조력 에이전트·코어 스킬·산출물 뷰어를 맞춤 생성한다. 마지막엔 첫 산출물 하나를 즉석에서 만들어 브라우저에 띄워 보여주고 물러난다.

03상태 ≠ 지식

변하는 구조화 데이터
상태 → workspace.db
업무 · 산출물 메타 · 결정 로그

SQLite 단일 파일. 쿼리·집계·대시보드용. Docker 없음.

쌓이는 텍스트 지식
지식 → wiki/*.md
정책 · 배운 것 · 결정의 이유

평문은 실패 모드가 없고, 띄울 서비스가 없고, git 으로 버전되고, 어떤 LLM이든 읽는다.

04생성되는 구조

my-workspace/
├── CLAUDE.md              # 정체성·규칙·조력자 (인터뷰로 생성)
├── PROGRAM.md             # 아이디어 파일 — 작동 원리 + 개선 로그
├── .claude/
│   ├── agents/            # 인터뷰가 낳은 조력자 1~2명
│   └── skills/            # /task · /deliver · /view
├── workspace.db           # [상태] SQLite — tasks · deliverables
├── wiki/                  # [지식] 플랫 마크다운 (RAG 없음)
│   ├── index.md
│   └── *.md               # 한 파일 = 한 주제
├── bin/
│   ├── ws.py              # DB 헬퍼 CLI (단일 파일, 의존성 0)
│   └── render.py          # md→html + 대시보드 빌더
├── deliverables/          # slug.md + slug.html 쌍
│   └── _dashboard.html    # 자동 생성 — 더블클릭으로 열림
├── notes/inbox/           # 메모 → 정제되면 wiki/ 로 승격
└── archive/
