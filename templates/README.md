# 산출물 템플릿 📑

`deliver` 로 산출물을 만들 때 골라 쓰는 시작 골격 모음입니다. 각 파일은 그냥 마크다운이라 **열어서 참고**하거나 **복제해서 나만의 것으로 고쳐** 써도 됩니다.

## 쓰는 법

```bash
python bin/ws.py templates                        # 사용 가능한 템플릿 목록
python bin/ws.py deliver "3분기 운영 보고" --template report   # report 골격으로 산출물 생성
```

`--template <이름>` 을 주면 그 골격을 본문으로 산출물이 생성되고, `{{title}}`·`{{date}}` 는 실제 값으로 자동 치환됩니다. (Claude Code 에게는 그냥 `/deliver` 로 "보고서 형식으로 정리해줘" 처럼 말해도 됩니다.)

## 카탈로그

| 이름 | 쓰임 |
|---|---|
| `report` | 운영·실적 보고서 (요약→지표→세부→다음) |
| `meeting` | 회의록 (참석→안건→논의→결정→액션) |
| `plan` | 기획·계획 (배경→목표→범위→일정→리스크) |
| `spec` | 기능·시스템 설계 스펙 (목적→요구→설계→엣지→검증) |
| `proposal` | 제안서 (한 문장 제안→문제→내용→근거→요청) |
| `decision` | 의사결정 기록 ADR (맥락→결정→이유→대안→결과) |
| `research` | 리서치·조사 (핵심 질문→발견→출처→결론) |
| `guide` | 안내·가이드 문서 (대상→준비물→단계→FAQ) |
| `checklist` | 체크리스트 (단계별 확인 항목) |
| `weekly` | 주간 업무 정리 (한 일→다음 주→막힌 것) |
| `retro` | 회고 KPT (지속/문제/시도 + 기간 숫자) |
| `postmortem` | 장애·사고 회고 (요약→영향→타임라인→원인→재발방지) |

## 나만의 템플릿 추가

`templates/<이름>.md` 파일을 하나 떨어뜨리면 끝입니다. 코드 변경 없이 바로 `--template <이름>` 으로 잡힙니다.

- 첫 줄에 `<!-- desc: 한 줄 설명 -->` 을 넣으면 `templates` 목록에 그 설명이 뜹니다.
- 본문 어디든 `{{title}}` · `{{date}}` 를 쓰면 생성 시 치환됩니다.
